PMS ML Service Plugin Guide

Build, package, and deploy custom machine learning models into the PMS platform — from cloud-based inference scripts to fully self-training models with custom data pipelines.

Introduction

The PMS ML Service is a production-grade model serving and training platform built on MLflow, Celery, and MinIO. It supports two distinct plugin types:

📦

Pre-trained Plugins

Deploy an existing model — from a local file, HuggingFace Hub, S3, Roboflow, or any cloud API. No training step required. Package as a ZIP archive and upload.

🧠

Self-Training Plugins

Write a BaseTrainer subclass with your own data pipeline, preprocessing, training loop, and evaluation logic. Upload the .py file — the platform handles scheduling, versioning, and serving.

Base URL All ML service endpoints are at http://<host>:8030/api/v1/

Architecture

Every plugin goes through the same pipeline regardless of type:

┌─────────────────────────────────────────────────────────────────┐

│  ml-service (FastAPI :8030)                                      │

└─────────────────────────────────────────────────────────────────┘

Pre-trained Plugins

A pre-trained plugin packages an already-trained model (or a call to a cloud ML API) into a ZIP archive that the platform can store, version, and serve. You define exactly how the model loads and runs inference — the platform handles everything else.

Key Insight The model_file in your manifest does NOT have to be a neural network weight file. It can be any file your inference script needs — a config.json, a label_map.json, or even a small SQLite database.

ZIP Structure

Both flat and nested layouts are supported. macOS Finder compress (which adds __MACOSX/) is handled automatically.

Flat Layout

Inference Script Contract

Your inference.py must define exactly one class that inherits from mlflow.pyfunc.PythonModel and implements two methods:

import mlflow.pyfunc

class MyModel(mlflow.pyfunc.PythonModel):

    def load_context(self, context):
        """
        Called ONCE when the model is loaded for the first time.
        Use this to:
          • Load the model weights / config
          • Resolve environment variables (API keys, endpoints)
          • Initialize expensive resources (DB connections, HTTP clients)

        context.artifacts["model_file"]  → absolute path to your model_file
        context.artifacts["my_artifact"] → absolute path to artifacts/my_artifact
        """
        import os
        self.threshold = 0.5
        self.api_key = os.environ.get("MY_API_KEY") or "default"
        # load your model here:
        # self.model = load_something(context.artifacts["model_file"])

    def predict(self, context, model_input):
        """
        Called on EVERY inference request.
        model_input: whatever the caller sends in "inputs" (dict, list, DataFrame, etc.)
        Returns: any JSON-serializable value (dict, list, str, number)
        """
        row = self._to_dict(model_input)
        # run inference...
        return {"result": "..."}

    def _to_dict(self, model_input) -> dict:
        """Helper — handles both dict and DataFrame inputs."""
        if hasattr(model_input, "to_dict"):
            return model_input.iloc[0].to_dict() if len(model_input) else {}
        return dict(model_input)

Environment Variable Pattern

Always resolve credentials in load_context, never at module level. Use or fallbacks so None doesn't silently pass through:

def load_context(self, context):
    import os
    # ✅ Correct — "or" fallback ensures None becomes the default
    self.api_key = os.environ.get("MY_API_KEY") or "fallback"

    # ❌ Wrong — get() returns None if var not set; boto3 ignores None
    # self.api_key = os.environ.get("MY_API_KEY", "fallback")

Artifacts & Extra Config Files

Any files placed in the artifacts/ directory of your ZIP are automatically registered and available in load_context:

# ZIP layout:
# my-model/manifest.json          model_file: "model.pt"
# my-model/model.pt
# my-model/artifacts/scaler.pkl
# my-model/artifacts/label_map.json

def load_context(self, context):
    import joblib, json

    self.model   = joblib.load(context.artifacts["model_file"])
    self.scaler  = joblib.load(context.artifacts["scaler"])        # key = filename without extension
    self.labels  = json.load(open(context.artifacts["label_map"]))
Artifact keys The key in context.artifacts is the filename without its extension. scaler.pkl → key "scaler", label_map.json → key "label_map".

import io, json, os, base64
import mlflow.pyfunc


class MyRoboflowModel(mlflow.pyfunc.PythonModel):

    def load_context(self, context):
        with open(context.artifacts["model_file"]) as f:
            cfg = json.load(f)
        self.api_url  = cfg["api_url"]
        self.model_id = cfg["model_id"]
        self.api_key  = os.environ.get("ROBOFLOW_API_KEY") or ""

    def predict(self, context, model_input):
        from inference_sdk import InferenceHTTPClient
        from PIL import Image

        row = self._to_dict(model_input)
        img_bytes = base64.b64decode(row["image_b64"])
        pil_img   = Image.open(io.BytesIO(img_bytes)).convert("RGB")

        client = InferenceHTTPClient(
            api_url=self.api_url,
            api_key=row.get("api_key") or self.api_key,
        )
        raw = client.infer(pil_img, model_id=self.model_id)
        return raw.get("predictions", [])

    def _to_dict(self, m):
        return m.iloc[0].to_dict() if hasattr(m, "to_dict") else dict(m)

import base64, io
import mlflow.pyfunc
import numpy as np


class MyDetector(mlflow.pyfunc.PythonModel):

    def load_context(self, context):
        from ultralytics import YOLO
        self.model = YOLO(context.artifacts["model_file"])

    def predict(self, context, model_input):
        import cv2
        row   = self._to_dict(model_input)
        conf  = float(row.get("threshold", 0.5))
        data  = base64.b64decode(row["image_b64"])
        arr   = np.frombuffer(data, np.uint8)
        img   = cv2.imdecode(arr, cv2.IMREAD_COLOR)

        results = self.model.predict(img, conf=conf, verbose=False)[0]
        detections = []
        for x1, y1, x2, y2, score, cls in results.boxes.data.tolist():
            detections.append({
                "bbox":  [int(x1), int(y1), int(x2), int(y2)],
                "class": int(cls),
                "score": round(float(score), 4),
            })
        return {"detections": detections, "count": len(detections)}

    def _to_dict(self, m):
        return m.iloc[0].to_dict() if hasattr(m, "to_dict") else dict(m)

import mlflow.pyfunc


class TabularClassifier(mlflow.pyfunc.PythonModel):

    def load_context(self, context):
        import joblib
        self.model  = joblib.load(context.artifacts["model_file"])
        self.scaler = joblib.load(context.artifacts["scaler"])

    def predict(self, context, model_input):
        import numpy as np
        row        = self._to_dict(model_input)
        features   = np.array([[row["f1"], row["f2"], row["f3"]]])
        scaled     = self.scaler.transform(features)
        pred       = self.model.predict(scaled)[0]
        proba      = self.model.predict_proba(scaled)[0].tolist()
        return {"class": int(pred), "probabilities": proba}

    def _to_dict(self, m):
        return m.iloc[0].to_dict() if hasattr(m, "to_dict") else dict(m)

Self-Training Plugins

A self-training plugin is a single Python file containing a class that inherits from BaseTrainer. The platform discovers the class, runs your training loop, logs the model to MLflow, and makes it available for inference — all automatically.

File Upload Upload your .py file via POST /api/v1/trainers/upload. The file is saved to the shared plugin volume and the trainer is registered immediately.

BaseTrainer Contract

from app.abstract.base_trainer import BaseTrainer, TrainingConfig, EvaluationResult, OutputFieldSpec, DerivedMetricSpec
from app.abstract.data_source import S3DataSource  # choose any data source


class MyTrainer(BaseTrainer):

    # ── Required class attributes ──────────────────────────────────────────────
    name        = "my-model"           # unique slug — becomes inference URL
    data_source = S3DataSource(        # where to load training data from
        bucket="pms-ml",
        key="datasets/my_dataset.csv",
    )

    # ── Optional class attributes ──────────────────────────────────────────────
    version     = "1.0.0"
    description = "My model description"
    framework   = "sklearn"             # sklearn | pytorch | tensorflow | custom
    schedule    = "0 2 * * *"           # cron — auto-retrain at 2am daily
    tags        = {"domain": "billing"}

    # ── UI display schema (see "UI Schema" section) ────────────────────────────
    output_display = [
        OutputFieldSpec("prediction", "label",      "Predicted Class", primary=True,  hint="Enter the correct class"),
        OutputFieldSpec("confidence", "confidence", "Confidence"),
    ]

    # ── Derived metrics for A/B comparison (see "UI Schema" section) ──────────
    derived_metrics = [
        DerivedMetricSpec("exact_match", "Exact Match Rate", unit="%", higher_is_better=True),
    ]

    # ── Required methods ───────────────────────────────────────────────────────

    def train(self, preprocessed_data, config: TrainingConfig):
        """
        Train your model. Return either:
          • the model object  (evaluation uses the default strategy)
          • (model, test_data) tuple  (platform calls self.evaluate(model, test_data))
        """
        from sklearn.ensemble import RandomForestClassifier
        model = RandomForestClassifier(n_estimators=100)
        X, y  = preprocessed_data
        model.fit(X, y)
        return model

    def predict(self, model, inputs):
        """
        Run inference at serve time. Called by the platform when
        POST /inference/{name} is received.
        """
        import numpy as np
        arr  = np.array([inputs["features"]])
        pred = model.predict(arr)[0]
        return {"prediction": int(pred)}

    # ── Optional methods ───────────────────────────────────────────────────────

    def preprocess(self, raw_data) -> tuple:
        """Transform raw data before train(). Default: identity."""
        import pandas as pd, numpy as np
        df = pd.read_csv(self._bytes_to_buf(raw_data))
        X  = df.drop("label", axis=1).values
        y  = df["label"].values
        return X, y

    def postprocess(self, predictions):
        """Transform raw predictions before returning to caller. Default: identity."""
        return predictions

    def evaluate(self, model, test_data) -> EvaluationResult:
        """Custom evaluation. Default: auto accuracy for sklearn classifiers."""
        X_test, y_test = test_data
        y_pred = model.predict(X_test)
        acc    = (y_pred == y_test).mean()
        return EvaluationResult(accuracy=acc, y_true=y_test.tolist(), y_pred=y_pred.tolist())

    def _bytes_to_buf(self, data):
        import io
        return io.BytesIO(data) if isinstance(data, bytes) else io.StringIO(data)

MLDock Dataset as Data Source

DatasetDataSource connects your trainer directly to an MLDock Dataset — a structured collection of field-based entries (images, files, text, numbers) gathered from collectors or uploaded by admins. It's the recommended data source for trainers that are tightly coupled to a specific collection pipeline.

Why use DatasetDataSource?

  • No S3 paths to manage — entries are stored and fetched automatically.
  • Schema-aware — each DatasetField maps to a column in the loaded data.
  • Live data — every training run fetches the latest entries; no stale CSV files.
  • Collector-driven — you can invite contributors via email to submit entries from mobile or desktop.
  • Empty-state guard — the Trainers page warns you when the dataset has no entries and lets you upload directly.

What data_source.load() returns

A list of dicts, one per entry, keyed by field label (lowercased, spaces→underscores):

[
    {"front_view": "https://…presigned-url…", "body_condition_score": "3.5"},
    {"front_view": "https://…presigned-url…", "body_condition_score": "2.0"},
    # …one dict per entry in the dataset
]

For image / file fields the value is a presigned S3 URL (valid 1 hr). Download in preprocess():

import requests, numpy as np
from PIL import Image
from io import BytesIO

def preprocess(self, raw_data):
    X, y = [], []
    for row in raw_data:
        resp = requests.get(row["front_view"])
        img = Image.open(BytesIO(resp.content)).resize((224, 224))
        X.append(np.array(img).flatten())
        y.append(float(row["body_condition_score"]))
    return np.array(X), np.array(y)

Populating the dataset

self._fetch_bytes — File Fetching Helper

When your dataset contains file or image fields, each raw entry dict has a file_key (S3 object key) and a file_url (presigned URL). Do not use requests.get(file_url) directly — presigned URLs embed the internal MinIO hostname (minio:9000) which may be inaccessible depending on network configuration.

BaseTrainer provides self._fetch_bytes(file_key, file_url) as a built-in static method. It tries to read from S3 directly first (reliable in all environments), and falls back to the presigned URL if S3 access fails.

Standard pattern — CSV / Excel from file field

import io
import pandas as pd

def preprocess(self, raw):
    # Prefer the "Clean Copy" field; fall back to "Original Upload"
    clean = [e for e in raw
             if e.get("field_label") == "Clean Copy"
             and (e.get("file_key") or e.get("file_url"))]
    orig  = [e for e in raw
             if e.get("field_label") == "Original Upload"
             and (e.get("file_key") or e.get("file_url"))]
    entry = (clean or orig)
    if not entry:
        raise ValueError("No file data in dataset — upload a CSV via the Datasets page.")

    e       = entry[-1]                               # most recent upload
    content = self._fetch_bytes(e.get("file_key"), e.get("file_url"))
    if content is None:
        raise ValueError("Could not fetch file from storage.")

    fname = (e.get("file_key") or "").split("/")[-1].lower()
    if fname.endswith((".xlsx", ".xls")):
        df = pd.read_excel(io.BytesIO(content))
    else:
        try:
            df = pd.read_csv(io.BytesIO(content))
        except Exception:
            # rows with unquoted commas / extra fields — skip bad lines
            df = pd.read_csv(io.BytesIO(content), on_bad_lines="skip", engine="python")

    return df
CSV with unquoted commas? If your source CSV has field values that contain commas and no quoting, the strict C parser will raise a ParserError: Expected N fields … saw M. The try/except above automatically re-parses with on_bad_lines='skip' and the Python engine, which is more tolerant of malformed rows.

import numpy as np

predict() & Inference Inputs

predict(self, model, inputs) is called at inference time on a fresh trainer instance. Any state stored on self during preprocess() or train() is lost.

The Golden Rule

Never store a fitted transformer on self and use it in predict(). The trainer object is re-instantiated for every inference request — self.scaler, self.encoder, self.feature_names, etc. set during training are all gone.

Pattern A — sklearn Pipeline (supervised tabular)

Scaler, encoder, and imputer are Pipeline steps. MLflow saves the entire pipeline as one joblib artifact. auto_train_tabular() already returns a Pipeline.

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import RandomForestClassifier

def train(self, preprocessed, config: TrainingConfig):
    X_train, y_train = preprocessed
    pipeline = Pipeline([
        ('scaler', StandardScaler()),
        ('clf',    RandomForestClassifier(n_estimators=200, random_state=config.random_seed)),
    ])
    pipeline.fit(X_train, y_train)
    return pipeline          # scaler + model saved together in MLflow artifact

def predict(self, model, inputs: dict) -> dict:
    import pandas as pd
    row  = pd.DataFrame([[inputs['f1'], inputs['f2']]], columns=['f1', 'f2'])
    pred = model.predict(row)[0]   # Pipeline applies scaler then predicts
    proba = model.predict_proba(row)[0].tolist()
    return {"label": str(pred), "confidence": round(max(proba), 4)}

Pattern B — file inputs (base64-encoded CSV or image)

When input_schema declares a field with "type": "file", the inference endpoint base64-encodes the uploaded bytes before passing them to predict(). Always decode with base64.b64decode().

def predict(self, model, inputs: dict) -> dict:
    import base64, io, pandas as pd

    # inputs['transaction_data'] is base64-encoded bytes of the uploaded CSV
    csv_bytes = base64.b64decode(inputs['transaction_data'])
    df = pd.read_csv(io.StringIO(csv_bytes.decode('utf-8')))

    preds = model.predict(df[self.get_feature_names()])
    return {"predictions": preds.tolist(), "count": len(preds)}

TrainerBundle — Multi-Artifact Container

When a sklearn Pipeline is not suitable — unsupervised models (KMeans, IsolationForest), multi-step preprocessing that does not fit the Pipeline API, or when you need to carry a fitted threshold or label map — return a TrainerBundle from train(). The platform saves the entire bundle as a single joblib artifact via mlflow.sklearn. 

from app.abstract.base_trainer import BaseTrainer, TrainerBundle, TrainingConfig

class AnomalyDetector(BaseTrainer):
    name      = "anomaly_detector"
    framework = "sklearn"

    def train(self, preprocessed, config: TrainingConfig):
        import numpy as np
        from sklearn.ensemble import IsolationForest
        from sklearn.preprocessing import StandardScaler

        feature_cols = list(preprocessed.columns)
        X = preprocessed.values.astype(float)

        scaler   = StandardScaler()
        X_scaled = scaler.fit_transform(X)

        iso = IsolationForest(contamination=0.05, random_state=config.random_seed)
        iso.fit(X_scaled)

        # Fit threshold on training scores so predict() is consistent
        scores    = iso.decision_function(X_scaled)
        threshold = float(np.percentile(scores, 5))

        return TrainerBundle(
            model=iso,
            scaler=scaler,
            feature_names=feature_cols,
            threshold=threshold,
        )

    def predict(self, model: TrainerBundle, inputs: dict) -> dict:
        import numpy as np
        row      = np.array([[inputs.get(f, 0) for f in model.feature_names]])
        X_scaled = model.scaler.transform(row)
        score    = float(model.model.decision_function(X_scaled)[0])
        is_anom  = score < (model.threshold or 0.0)
        return {"is_anomaly": is_anom, "anomaly_score": round(score, 4)}

How it is saved — When train() returns a TrainerBundle, the platform automatically calls mlflow.sklearn.log_model(bundle, ...) regardless of the trainer's framework attribute. joblib serialises the entire bundle (model + scaler + all fields) into one artifact. _load_raw_model() at inference time loads it back and hands the whole bundle to trainer.predict(model, inputs).

Training Lifecycle

When POST /training/start is called, the platform executes this sequence:

Load data

data_source.load() is called. Returns raw bytes, string, list, or any format your data source returns.

Preprocess

trainer.preprocess(raw_data) transforms raw data into whatever train() expects (numpy arrays, DataFrames, PyTorch datasets, etc.). Override this — defaults to identity.

Train

trainer.train(preprocessed_data, config) returns the trained model (or a (model, test_data) tuple if you want custom evaluation).

Evaluate

trainer.evaluate(model, test_data) returns an EvaluationResult. Metrics are logged to MLflow automatically.

Log to MLflow

The model is saved to MLflow with the appropriate flavour (mlflow.sklearn, mlflow.pytorch, etc.) and registered under trainer.name.

Create deployment record

A ModelDeployment is written to MongoDB with status: "active", making the model immediately available for inference.

Example: Rent Prediction (sklearn Pipeline — scaler embedded)

This example uses a sklearn Pipeline so the StandardScaler is saved inside the MLflow artifact. predict() calls model.predict(row) — the pipeline applies scaling automatically. No self.scaler needed. 

import io
from app.abstract.base_trainer import BaseTrainer, TrainingConfig, EvaluationResult
from app.abstract.data_source import S3DataSource


_FEATURES = ["bedrooms", "bathrooms", "sq_ft", "floor"]


class RentPredictionTrainer(BaseTrainer):
    name        = "rent-predictor"
    version     = "1.0.0"
    description = "Predicts monthly rent from property features"
    framework   = "sklearn"
    schedule    = "0 3 * * 1"   # retrain every Monday at 3am

    data_source = S3DataSource(bucket="pms-ml", key="datasets/properties.csv")

    def preprocess(self, raw_data):
        import pandas as pd
        df = pd.read_csv(io.BytesIO(raw_data))
        return df[["bedrooms", "bathrooms", "sq_ft", "floor", "monthly_rent"]]

    def train(self, preprocessed, config: TrainingConfig):
        from sklearn.pipeline import Pipeline
        from sklearn.preprocessing import StandardScaler
        from sklearn.ensemble import GradientBoostingRegressor
        from sklearn.model_selection import train_test_split

        X = preprocessed[_FEATURES].values
        y = preprocessed["monthly_rent"].values
        X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=config.random_seed)

        # Scaler is a Pipeline step — saved with the model in the MLflow artifact.
        # predict() calls model.predict(X) and scaling is applied automatically.
        pipeline = Pipeline([
            ('scaler', StandardScaler()),
            ('gbr',    GradientBoostingRegressor(n_estimators=200, learning_rate=0.05,
                                                    max_depth=5, random_state=config.random_seed)),
        ])
        pipeline.fit(X_tr, y_tr)
        return pipeline, (X_te, y_te)   # tuple → platform calls evaluate(pipeline, (X_te, y_te))

    def evaluate(self, model, test_data) -> EvaluationResult:
        from sklearn.metrics import mean_absolute_error, r2_score
        X_te, y_te = test_data
        y_pred = model.predict(X_te)   # pipeline: scale → predict
        return EvaluationResult(
            mae=float(mean_absolute_error(y_te, y_pred)),
            r2=float(r2_score(y_te, y_pred)),
            y_true=y_te.tolist(), y_pred=y_pred.tolist(),
        )

    def predict(self, model, inputs: dict) -> dict:
        import numpy as np
        row  = np.array([[inputs.get(f, 0) for f in _FEATURES]])
        pred = float(model.predict(row)[0])   # pipeline: scale → predict (no self.scaler!)
        return {"monthly_rent": round(pred, 2)}

    def get_feature_names(self):
        return _FEATURES

import io
from app.abstract.base_trainer import BaseTrainer, TrainerBundle, TrainingConfig, EvaluationResult
from app.abstract.data_source import DatasetDataSource


_FEATURES = ["monthly_spend", "purchase_frequency", "avg_order_value",
             "days_since_last_purchase", "login_count_30d"]


class CustomerSegmentation(BaseTrainer):
    name      = "customer_segmentation"
    framework = "sklearn"

    data_source = DatasetDataSource(
        slug="customer-data",
        auto_create_spec={
            "name": "Customer Data",
            "fields": [{"label": "Original Upload", "type": "file", "required": True}],
        },
    )

    input_schema = {
        "monthly_spend":            {"type": "number", "label": "Monthly Spend (KES)", "required": True},
        "purchase_frequency":       {"type": "number", "label": "Purchases / Month",    "required": True},
        "avg_order_value":          {"type": "number", "label": "Avg Order (KES)",       "required": True},
        "days_since_last_purchase": {"type": "number", "label": "Days Since Purchase",   "required": True},
        "login_count_30d":          {"type": "number", "label": "Logins (30 days)",      "required": False},
    }

    def preprocess(self, raw):
        import pandas as pd
        entry = next((e for e in raw
                      if e.get("field_type") == "file"
                      and (e.get("file_key") or e.get("file_url"))), None)
        if not entry:
            raise ValueError("No file in dataset")
        content = self._fetch_bytes(entry.get("file_key"), entry.get("file_url"))
        df = pd.read_csv(io.BytesIO(content))
        cols = [c for c in _FEATURES if c in df.columns]
        return df[cols].fillna(df[cols].median())

    def train(self, preprocessed, config: TrainingConfig):
        import numpy as np
        from sklearn.preprocessing import StandardScaler
        from sklearn.cluster import KMeans

        feature_cols = list(preprocessed.columns)
        X = preprocessed.values.astype(float)

        scaler   = StandardScaler()          # ← fitted on training data
        X_scaled = scaler.fit_transform(X)

        kmeans = KMeans(n_clusters=4, n_init=20, random_state=config.random_seed)
        kmeans.fit(X_scaled)

        # Assign labels by ascending mean monthly_spend per cluster
        spend_idx = feature_cols.index("monthly_spend") if "monthly_spend" in feature_cols else 0
        spend_by_c = {c: X[kmeans.labels_ == c, spend_idx].mean() for c in range(4)}
        sorted_c   = sorted(spend_by_c, key=lambda c: spend_by_c[c])
        names      = ["Dormant", "Occasional", "Regular", "High Value"]
        label_map  = {int(c): names[i] for i, c in enumerate(sorted_c)}

        # TrainerBundle bundles model + scaler + feature_names + label_map into
        # one joblib artifact.  predict() receives the loaded bundle as `model`.
        return TrainerBundle(
            model=kmeans,
            scaler=scaler,          # ← saved, not stored on self
            feature_names=feature_cols,
            label_map=label_map,
        )

    def predict(self, model: TrainerBundle, inputs: dict) -> dict:
        import numpy as np
        row      = np.array([[inputs.get(f, 0) for f in model.feature_names]])
        # model.scaler is the fitted StandardScaler from train() — loaded from MLflow
        X_scaled = model.scaler.transform(row)
        cluster  = int(model.model.predict(X_scaled)[0])
        label    = model.label_map.get(cluster, str(cluster))
        dist     = float(np.linalg.norm(X_scaled - model.model.cluster_centers_[cluster]))
        return {"segment_label": label, "segment_id": cluster, "distance": round(dist, 4)}

import torch, torch.nn as nn
from app.abstract.base_trainer import BaseTrainer, TrainingConfig
from app.abstract.data_source import S3DataSource


class SimpleNet(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_features, 128), nn.ReLU(),
            nn.Linear(128, 64),          nn.ReLU(),
            nn.Linear(64, out_features),
        )
    def forward(self, x):
        return self.net(x)


class TenantChurnTrainer(BaseTrainer):
    name       = "tenant-churn"
    framework  = "pytorch"
    schedule   = "0 4 * * *"
    data_source = S3DataSource(bucket="pms-ml", key="datasets/churn.parquet")

    def preprocess(self, raw_data):
        import pandas as pd, io
        from sklearn.model_selection import train_test_split
        df = pd.read_parquet(io.BytesIO(raw_data))
        X  = df.drop("churned", axis=1).values.astype("float32")
        y  = df["churned"].values.astype("float32")
        return train_test_split(X, y, test_size=0.2)

    def train(self, preprocessed_data, config: TrainingConfig):
        X_train, X_test, y_train, y_test = preprocessed_data
        device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

        model = SimpleNet(in_features=X_train.shape[1], out_features=1).to(device)
        opt   = torch.optim.Adam(model.parameters(), lr=1e-3)
        loss_fn = nn.BCEWithLogitsLoss()

        X_t = torch.tensor(X_train).to(device)
        y_t = torch.tensor(y_train).unsqueeze(1).to(device)

        for epoch in range(config.max_epochs):
            model.train()
            opt.zero_grad()
            loss = loss_fn(model(X_t), y_t)
            loss.backward()
            opt.step()

        return model, (X_test, y_test)

    def predict(self, model, inputs: dict):
        import torch
        x = torch.tensor([inputs["features"]], dtype=torch.float32)
        with torch.no_grad():
            prob = torch.sigmoid(model(x)).item()
        return {"churn_probability": round(prob, 4), "will_churn": prob > 0.5}

auto_train_tabular — Zero-boilerplate Tabular Training

The most powerful shortcut in BaseTrainer. Pass a DataFrame with a label column and it handles feature detection, scaling, encoding, train/val/test splitting, model selection, cross-validation, and MLflow logging automatically.

class MyTabularTrainer(BaseTrainer):
    name      = "my-tabular-model"
    framework = "sklearn"
    data_source = S3DataSource(bucket="pms-ml", key="datasets/data.csv")

    def preprocess(self, raw_data):
        import pandas as pd, io
        return pd.read_csv(io.BytesIO(raw_data))   # just return the DataFrame

    def train(self, preprocessed, config: TrainingConfig):
        config.task       = "classification"   # or "regression"
        config.test_split = 0.2
        config.val_split  = 0.1
        return self.auto_train_tabular(preprocessed, label_col="target", config=config)

    def predict(self, model, inputs: dict):
        import pandas as pd
        # Always pass a named DataFrame — avoids column-selector issues
        row = {name: float(inputs.get(name, 0)) for name in self.get_feature_names()}
        arr = pd.DataFrame([row])
        pred = model.predict(arr)[0]
        proba = model.predict_proba(arr)[0].tolist() if hasattr(model, "predict_proba") else [float(pred)]
        return {"prediction": int(pred), "probabilities": proba}

What auto_train_tabular does automatically

🔍

Feature detection

Separates numeric and categorical columns automatically. Uses integer column indices internally so inference works with both DataFrames and numpy arrays.

⚙️

Preprocessing pipeline

StandardScaler for numerics, OrdinalEncoder for categoricals — all wrapped in an sklearn Pipeline so scaling is never applied to test data before training.

🤖

Model selection

Tries RandomForest, GradientBoosting, LogisticRegression / Ridge. Uses config.task to pick classifier vs regressor variants. Returns the best model.

📊

Evaluation

Calls your evaluate() override if present, otherwise auto-evaluates with accuracy+F1+ROC-AUC (classification) or MAE+R² (regression). All metrics logged to MLflow.

predict() must use DataFrame, not numpy array Always pass pd.DataFrame([row], columns=feature_names) in your predict() — the sklearn Pipeline uses column positions internally, and a named DataFrame is the only safe way to ensure ordering stays correct.

auto_train_torch — GPU-accelerated PyTorch Training

Drop-in training loop for PyTorch models. Handles device selection, AMP mixed precision, gradient clipping, LR scheduling, and early stopping.

import torch.nn as nn
from app.abstract.base_trainer import BaseTrainer, TrainingConfig


class MyNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(10, 128), nn.ReLU(), nn.Dropout(0.3),
            nn.Linear(128, 64),  nn.ReLU(),
            nn.Linear(64, 1),
        )

    def forward(self, x): return self.layers(x)


class MyTorchTrainer(BaseTrainer):
    name      = "my-torch-model"
    framework = "pytorch"

    def preprocess(self, raw_data):
        import pandas as pd, numpy as np, io
        from sklearn.model_selection import train_test_split
        df = pd.read_csv(io.BytesIO(raw_data))
        X  = df.drop("target", axis=1).values.astype("float32")
        y  = df["target"].values.astype("float32")
        return train_test_split(X, y, test_size=0.2)

    def train(self, preprocessed, config: TrainingConfig):
        X_train, X_test, y_train, y_test = preprocessed
        model = MyNet()

        # auto_train_torch handles device, AMP, scheduler, early stopping
        trained = self.auto_train_torch(
            model=model,
            train_data=(X_train, y_train),
            val_data=(X_test, y_test),   # optional — enables early stopping
            config=config,
            loss_fn="bce",              # "bce" | "ce" | "mse" | "mae" | custom nn.Module
        )
        return trained, (X_test, y_test)

Security System

The ML service includes a multi-layer security system that logs every request, detects malicious IPs using heuristics + ML, and scans uploaded files for threats.

🛡️

Request Logger Middleware

Every request is logged to MongoDB with IP, method, path, headers (credentials redacted), payload size, latency, and status. Banned IPs are rejected before the handler runs.

🔍

IP Threat Detection

Heuristic scoring (rate limits, error rate, suspicious paths, user-agent) blended 40/60 with an ML model trained on live request data. IPs scoring ≥ 0.90 are auto-banned.

🦠

File Scanner

Two-stage pipeline: ClamAV virus scan → static code analysis (regex + AST). All trusted developer uploads (trainer .py files and model ZIP archives) get ClamAV-only — static analysis is skipped because legitimate ML code has too many false positives.

🤖

IP Threat Trainer

Self-training model (ip_threat_detector) that retrains every 30 minutes from live ip_records data. Bootstraps from synthetic data when fewer than 20 real records exist.

🔒

Runtime Audit Hook

CPython's sys.addaudithook is installed in every Celery worker process before any task runs. Blocks subprocess.Popen (unless whitelisted), os.system, and sensitive file opens. Logs unexpected socket connections. Irreversible — user code cannot remove it.

🌐

SafeHttpClient

Monitored httpx wrapper for use inside inference scripts and trainers. Enforces a host allowlist, 50 MB response cap, and logs every request with URL, status, latency, and response size. Raises HostNotAllowedError before any connection is made to a non-whitelisted host.

Runtime Audit Hook

The worker installs a sys.addaudithook into every Celery ForkPoolWorker process before the first task runs. CPython calls this hook for every auditable event — subprocess spawning, file opens, socket connections — regardless of where the call originates in the call stack.

Irreversible by design sys.addaudithook cannot be removed once installed. User code — including injected inference scripts and trainer plugins — cannot uninstall, replace, or silence it. This is CPython's intended security primitive for exactly this use case.

Policy table

Audit eventActionDetails
subprocess.Popen BLOCK Raises PermissionError unless the executable basename is in the allowlist. Covers all subprocess.run/call/check_output/Popen calls.
os.system BLOCK Always blocked — no allowlist. Use subprocess.run with a whitelisted executable instead.
open (sensitive paths) BLOCK Raises PermissionError for /etc/passwd, /etc/shadow, /etc/sudoers, /root/.ssh/, /etc/cron*, /proc/<pid>/mem|maps|environ.
socket.connect LOG Logs a WARNING (host, port) for connections to hosts not in the internal allowlist. Does not raise — framework code (boto3, motor, MLflow) also connects and must not be broken. Use iptables for hard network enforcement.

Subprocess executable allowlist

Only the following basenames are permitted as subprocess arguments. Everything else — curl, wget, nc, bash -c, arbitrary binaries — is blocked immediately.

Layered defence — what the hook does not replace

The audit hook is a Python-level control. It does not intercept syscalls made by C extensions (e.g. a compiled PyTorch kernel writing a file directly). For defence-in-depth, combine it with:

  • iptables / network namespace — hard-block all worker egress except to internal services
  • seccomp profile — restrict allowed syscalls at the kernel level
  • ClamAV — signature-based scan before any code reaches the worker
  • SafeHttpClient — allowlist-enforced HTTP for outbound API calls inside scripts

Data Drift Detection

Data drift occurs when the distribution of inference inputs changes significantly from when the model was trained (or baselined). The drift service detects this automatically and fires notifications so you know when to retrain.

How it works

Set a baseline

Call POST /monitoring/drift/{trainer}/baseline once the model has accumulated enough inferences (default: 500 most recent successful calls). The service computes per-feature statistics and stores them in drift_baselines.

Scheduled drift checks

Every 6 hours the scheduler calls drift_service.run_all_drift_checks() for every trainer that has a baseline. It compares the last 200 successful inferences against the baseline distribution.

Alert raised & SSE event published

If any feature exceeds its threshold, a DriftAlert document is written and a drift_alert SSE event is published on the ml:events Redis channel.

Review & resolve

Query alerts via GET /monitoring/drift/{trainer}/alerts. Acknowledge or resolve via PATCH /monitoring/drift/alerts/{id}. Re-train the model, then refresh the baseline.

Detection methods

Feature typeMethodDefault thresholdWhen triggered
Numeric KS test (scipy) or Z-score (fallback) KS ≥ 0.20 | |Z| ≥ 2.5 Shape of distribution has shifted (not just mean)
Categorical PSI — Population Stability Index PSI ≥ 0.20 Class proportions have meaningfully shifted (0.1–0.2 = moderate; >0.2 = significant)
scipy is optional If scipy is not installed the KS test falls back to a Z-score mean-shift comparison. Install scipy for full two-sample KS testing: pip install scipy.

Which features are tracked

Only scalar input fields from the inference inputs dict are tracked. Fields are automatically excluded if:

  • The key ends with _b64 or _key (binary/image data)
  • The key is file_b64
  • The string value is longer than 256 characters
  • Fewer than 30 samples were observed for that feature

A/B Testing

A/B testing in ML Studio compares two or more versions of the same model against identical inputs. Version A and version B are both deployments of the same trainer — B is selected from the same model's version history, not from a different model entirely.

Key mental model You train your model, collect feedback, improve the training data or hyper-parameters, retrain — and now you have v1 and v2 of the same model. A/B testing answers: "Is v2 actually better than v1 on real production inputs?"
🅰

Version A — the baseline

Your current production deployment. Already has accumulated feedback from real users — EMR, digit accuracy, billing impact are all computed from this history.

🅱

Version B — the challenger

A newer deployment of the same trainer. Selected from the same model's version list. May have little or no feedback yet — live output comparison fills the gap.

Not cross-model comparison A/B testing is not "compare WaterMeterOCR vs PlateDetector". It is "compare WaterMeterOCR v1 vs WaterMeterOCR v2". Both versions share the same trainer_name, the same input schema, and the same derived_metrics spec — making the comparison meaningful.

How it works

Open any model in ML Studio → Compare tab. The panel shows all deployments of that trainer, split into two explicit pickers.

Pick version A — the baseline

Select one version from the A — Baseline column. This is the version you are measuring against — typically the current production deployment. It defaults to whichever deployment is currently open.

The selected version gets a filled ● dot and a blue version label. All other versions in the B column remain selectable.

Pick version B — the challenger

Select one version from the B — Challenger column. This must be a different deployment of the same trainer — not a different model. The version chosen as A is automatically disabled in the B list (shown greyed with a note), preventing the same version from being picked twice.

If the two versions have different input schemas a warning banner appears — results may not be directly comparable.

Supply test inputs

The JSON inputs from your last inference run are pre-populated. Edit to use a representative real-world sample — not a toy example. Use an input that exercises a known-tricky case (e.g. a blurry meter image) for the most meaningful comparison.

Run — three result areas appear

Click Compare A vs B. Both deployments run in parallel with identical inputs. Three areas render:

Trainer Metrics — not just accuracy

The Trainer Metrics table shown at the top of every A/B comparison is entirely determined by what the trainer declares in derived_metrics. It is not a generic accuracy score — it is whatever set of measurements makes sense for that model's domain. A classifier, an OCR model, and a regression model all have different meaningful measures.

Why trainer-declared, not platform-default? The platform cannot know whether a difference of 1 in a meter reading is serious (a 1-unit billing error) or irrelevant (a 1-digit reading confidence score). Only the trainer author knows. Declaring derived_metrics puts that domain knowledge directly in the code, next to the model.

Accuracy is not enough for OCR

For a water meter reader, a binary "correct / incorrect" accuracy misses critical nuance. A reading of 04823 vs 04924 is "wrong" in both cases, but one has an edit distance of 1 (one digit off) while the other might have an edit distance of 5 (completely wrong) — and the billing impact is wildly different. Declare all four metrics: 

derived_metrics = [
    # Binary: was the full reading correct?
    DerivedMetricSpec("exact_match",    "Exact Match Rate",
                      description="% of readings 100% correct",
                      unit="%", higher_is_better=True),

    # Partial credit: how many digit positions are right
    # "04823" vs "04924" → 3/5 = 60% digit accuracy
    DerivedMetricSpec("digit_accuracy", "Digit Accuracy",
                      description="Per-character digit position accuracy",
                      unit="%", higher_is_better=True),

    # String edit distance: distance("04823", "04924") = 2
    DerivedMetricSpec("edit_distance",  "Edit Distance",
                      description="Mean Levenshtein distance (lower = better)",
                      unit="chars", higher_is_better=False),

    # Business impact: abs(4823 - 4924) = 101 units over-billed
    DerivedMetricSpec("numeric_delta",  "Billing Impact",
                      description="Mean abs(predicted − actual) in meter units",
                      unit="units", higher_is_better=False),
]

Metrics per domain

DomainRecommended metricsWhy
Water / electricity meter OCR exact_match, digit_accuracy, edit_distance, numeric_delta Binary accuracy ignores partial correctness; billing impact is the real business cost
Document / text OCR exact_match, edit_distance Edit distance captures near-misses that binary correct/wrong misses
Classification (low cardinality) exact_match Binary is sufficient; confusion matrix in the Feedback tab provides per-class detail
Classification (high cardinality) exact_match, edit_distance High-cardinality label spaces benefit from string similarity as a proxy for semantic closeness
Numeric regression numeric_delta Mean absolute error in the problem's native unit (KES, kg, kWh) is the most interpretable
Object detection / counting exact_match, numeric_delta Exact count match + mean count error tell you both precision and operational impact

This is by design: run the model in production, collect feedback, then return to the Compare tab to see statistically grounded numbers. The Live Output cards and Numeric Output Fields table (from the parallel run against your test input) are always available immediately regardless of feedback history.

CSV ParserError — safe read pattern

Real-world CSVs frequently contain unquoted commas inside fields (addresses, product descriptions, free-text notes). The default pandas C parser raises ParserError: Expected N fields in line X, saw N+1 when it encounters them. Always use the try/except fallback below — it applies everywhere you call pd.read_csv() in a trainer. 

# ❌ WRONG — crashes on CSVs with unquoted commas or extra fields:
df = pd.read_csv(io.BytesIO(content))

# ✅ CORRECT — copy this exact pattern for every pd.read_csv call:
try:
    df = pd.read_csv(io.BytesIO(content))
except Exception:
    df = pd.read_csv(io.BytesIO(content), on_bad_lines='skip', engine='python')

# For Excel files — no fallback needed (openpyxl is forgiving):
df = pd.read_excel(io.BytesIO(content))

engine='python' is slower but tolerates malformed rows. on_bad_lines='skip' drops rows that can't be parsed rather than aborting. For training data this trade-off is almost always correct — a few bad rows are preferable to a failed training run.

def predict(self, model, inputs):

def train(self, preprocessed, config):
    pipeline = Pipeline([('scaler', StandardScaler()), ('clf', RandomForestClassifier())])
    pipeline.fit(X, y)
    return pipeline   # scaler is inside the MLflow artifact

# ✅ OPTION B — TrainerBundle (unsupervised / multi-artifact):
from app.abstract.base_trainer import TrainerBundle

def train(self, preprocessed, config):
    scaler = StandardScaler()
    X_scaled = scaler.fit_transform(X)
    kmeans = KMeans(n_clusters=4).fit(X_scaled)
    return TrainerBundle(model=kmeans, scaler=scaler, feature_names=cols)

def predict(self, model: TrainerBundle, inputs):
    row = np.array([[inputs[f] for f in model.feature_names]])
    X_scaled = model.scaler.transform(row)   # scaler loaded from MLflow bundle
    cluster = int(model.model.predict(X_scaled)[0])
    return {"segment": cluster}
